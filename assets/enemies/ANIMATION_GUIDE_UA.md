# Схема анімації героїв і ворогів

## Повний аркуш `12x4`

Рядки:

1. рух вниз;
2. рух ліворуч;
3. рух праворуч;
4. рух вгору.

Стовпці:

1. `Idle 1`;
2. `Idle 2`;
3. `Move 1`;
4. `Move 2`;
5. `Move 3`;
6. `Move 4`;
7. `Attack 1`;
8. `Attack 2`;
9. `Attack 3`;
10. `Hurt`;
11. `Death 1`;
12. `Death 2`.

Для нових великих ворогів аркуш має розмір `576x256`, а один кадр —
`48x64`. Початкові герої та вороги використовують кадри `32x32`.

## Вибір напрямку

Аніматор отримує різницю між старою та новою позицією персонажа.

- `dx < 0` означає рух ліворуч;
- `dx > 0` означає рух праворуч;
- `dy < 0` означає рух вниз;
- `dy > 0` означає рух вгору.

Під час діагонального руху використовується напрямок сильнішої осі.
Якщо персонаж зупинився, зберігається останній напрямок і запускається
анімація `Idle`.

Кадри ходьби перемикаються за реально пройденою відстанню, а не лише
за часом. Тому повільний ворог не перебирає ногами так само швидко, як
герой під час спринту. Коротка затримка зміни напрямку прибирає
тремтіння на кутах маршруту.

## Дзеркальне відображення

Якщо в аркуші є всі чотири напрямки, використовується
`SpriteSheetLayout.fourDirections()`.

Якщо намальовані тільки рядки `Down`, `Left`, `Up`, використовується
`SpriteSheetLayout.threeDirectionsMirrored()`. У такому випадку рух
праворуч автоматично малюється як дзеркальна копія руху ліворуч.

Не треба дзеркалити кадри, якщо персонаж має несиметричну зброю, одяг
або окремо намальований правий ряд.

## Підключення до Entity

У конструкторі героя або ворога:

```java
setAnimator(new SpriteSheetAnimator(
    texture,
    32,
    64,
    24f,
    48f,
    SpriteSheetLayout.fourDirections()
));
```

Після зміни координат:

```java
updateAnimation(deltaTime, x - previousX, y - previousY);
```

Для окремих дій:

```java
playAttackAnimation();
playHurtAnimation();
playDeathAnimation();
```

Текстури повинні використовувати фільтр `Nearest`, щоб пікселі не
розмивалися під час масштабування.
